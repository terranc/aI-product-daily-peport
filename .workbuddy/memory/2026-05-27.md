### GitHub Pages 部署方式修正 ✅

**问题**: 切换分支后 docs/ 目录被清除，导致 cp 失败

**正确方式**:
1. 在 main 分支上将 docs/ 内容复制到临时目录 `/tmp/gh-pages-deploy`
2. 在临时目录初始化 git 仓库
3. 推送到 gh-pages 分支

**命令**:
```bash
/bin/rm -rf /tmp/gh-pages-deploy && /bin/mkdir -p /tmp/gh-pages-deploy && /bin/cp -r /Volumes/EXTEND/aI-product-daily-peport/docs/* /tmp/gh-pages-deploy/
cd /tmp/gh-pages-deploy && /opt/homebrew/bin/git init && /opt/homebrew/bin/git add -A && /opt/homebrew/bin/git commit -m "Deploy docs"
/opt/homebrew/bin/git remote add origin git@github.com:terranc/aI-product-daily-peport.git
/opt/homebrew/bin/git push -f origin main:gh-pages
```
