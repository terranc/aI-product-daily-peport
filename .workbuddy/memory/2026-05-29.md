- 原因：`rebaseModalUrls` 函数在产品详情页使用当前页面 URL 作为基准解析相对路径
- 修复：当 baseUrl 指向产品详情页时，使用网站根目录作为解析基准
- 结果：未生效，问题仍在

### 第二次修复（已生效）
- 根因：`fetch(detailUrl)` 使用相对路径，在产品详情页点击时被错误解析
- 修复：在 `openProductModal` 开头将相对路径转为绝对路径
  ```javascript
  const absoluteUrl = new URL(detailUrl, window.location.origin + '/').href;
  detailUrl = absoluteUrl;
  ```
- 部署问题：gh-pages 分支需要将 docs 内容放在根目录，而非 docs 子目录
- 已验证线上修复生效
